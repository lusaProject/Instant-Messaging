# 即时通讯（Instant Messaging, IM）简介

即时通讯（IM）是一种基于互联网的通信技术与服务模式，使用户能够在不同设备间以接近实时的方式交换信息。它不仅提升了沟通的效率，也改变了人们的社交、工作与协作方式。随着移动互联网与智能终端的普及，IM 已成为现代数字生活中最核心的通信手段之一。

IM 的最大特点在于低延迟、强互动、多媒体化和高度在线化，这使其广泛应用于社交平台、企业协作、在线客服、教育教学、物联网通知等各类场景。

---

# 核心功能与特点

## 1. 文本聊天
- 支持快速、轻量化的文字消息交换。
- 支持表情符号（Emoji）、富文本格式、高亮、代码片段等内容增强形式。
- 具备消息已读/未读状态、消息撤回、编辑等机制，确保沟通更灵活。

## 2. 多媒体传输
- 支持发送图片、音频、视频、文件、位置信息等多种媒体类型。
- 部分 IM 系统内置压缩与预览能力，提高传输效率与体验。
- 提供文件云端存储、历史文件检索等辅助功能。

## 3. 语音与视频通话
- 支持点对点（P2P）模式与多方会议模式。
- 利用 WebRTC、VoIP、回声消除、弱网优化等技术，提供清晰、稳定的通话质量。
- 支持屏幕共享、白板协作等扩展能力（在企业 IM 中尤为常见）。

## 4. 群组聊天
- 支持创建多人群组，实现团队讨论、活动组织、协作办公等需求。
- 群组可提供管理员权限、群公告、群文件、投票等管理工具。
- 大型 IM 系统支持超大群或社区（如数万人规模）。

## 5. 状态与在线显示
- 展示在线、离线、忙碌、勿扰、隐身等不同状态。
- 部分系统支持自定义状态与状态自动同步，例如“手机电量低”“会议中”等信息。

## 6. 通信协议
- 常用协议包括 WebSocket、MQTT、XMPP、SIP、HTTP/2、QUIC 等。
- 许多现代 IM 系统采用 WebSocket 协议以获取更低延迟。

## 7. 消息可靠性机制
- **消息投递保证**：确保消息送达、去重、防丢失。
- **离线消息同步**：用户离线后重新登录可获取未读消息。
- **多设备同步**：在手机、电脑、平板等多端保持聊天记录一致。

## 8. 存储与历史记录
- 消息通常存储在云端，以便跨设备同步。
- 大规模 IM 系统会使用分布式数据库、冷热存储分离、消息分片等技术。

## 9. 安全与隐私保护
- 使用 TLS/SSL 加密保障传输安全。
- 支持端到端加密（E2EE），确保只有通信双方可读取内容。
- 提供权限控制、垃圾信息过滤、用户举报等机制保护用户安全。
